## 🧠 Prompt Engineering Notes

This output was generated using a **structured, role-specific mega-prompt** that:

- Grounded Claude in a specific persona: *Technical Recruiter + Career Coach + Resume Writer + Personal Branding Consultant*
- Provided the target job description (Zoo Finance) alongside the resume
- Requested **12 named sections** with explicit formatting and length constraints
- Added a constraint to **use only information available in the resume** — no fabrication
- Asked for a **bonus section**: 5 target company recommendations based on actual skills

### What Made the Output Strong

1. **Specific anchoring** — the JD and resume together gave Claude concrete signals to match against
1. **Quantified achievements** in the resume (MAPE, accuracy, data scale) transferred directly to output
1. **Domain overlap** (Crypto AI System + Zoo Finance Web3 focus) enabled highly specific cover letter language
1. **No hallucination guard** — Claude was instructed to pull from the resume only, not invent

## 💡 Key Insight from This Experiment

> Job hunting forces you to do things you wouldn’t do otherwise — cold emailing, LinkedIn outreach, sending messages to recruiters. The surprising part wasn’t that Claude could write a cover letter. It was that it generated **the whole stack** — every channel of outreach, every angle of positioning — in one session, from one document.
> 
> The best prompts don’t ask Claude to answer questions. They give it a role, a context, constraints, and a target — and let it build.
